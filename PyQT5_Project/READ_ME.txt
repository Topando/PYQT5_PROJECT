ФАЙЛЫ py:

starter - запуск
Class - действия с классами, сделана работа интерфейса
Handler - функции без db
DBController - функции с db, обращение и тд
Variables - некоторые константы 

FILE:
file/name_task.txt - промежуточная переменная с номером задания, таким же как и db

БАЗЫ ДАННЫХ:

Task.db 
1. Option - только за вариант
2. Statistics - статистика при решении обычных задач
3. TaskAnswer - все задачи которые доступны

ПАПКИ:

Interface - Файлы ui и py интерфейса 

Task-Res:
 
Task - задания
Res - решения  		ОБЯЗАТЕЛЬНО добавлять по инструкции: имя файла совпадает в двух папках(если в Task 1.2 то и в Res 1.2),
			потом еще в самой программе надо добавить с ответом

ПО РАБОТЕ(Реализованный функционал):

Задания - вывод выбранного задания
Вариант - полный вариант, с результатом в конце
Статистика - вывод статистики(из Заданий)
Настройки - Добавить, удалить задания, сброс статистики

САМЫЕ ГЛАВНЫЕ ФУНКЦИИ:
